"In the beginner's mind there are many possibilities, in the expert's there are few."

mushin allows you to develop domain-specific ruby applications, along with domain experts via domain-specific languages, and domain extenstions.

mushin is complete awesomeness!

Welcome to Mushin

 $ mushin new domain

generate gem packageable domain-specific frameworks!

 $ mushin new ext

generate gem packageable domain-specific extensions!


Quick Start

$ gem install mushin
$ mushin --help


Philosophy

Software Engineers often need to utilize knowledge from more than one domain for developing a single application, where a domain is an area of knowledge such as an industry field or a subject of interest. Knowledge utilized within an application may belong to its core domain, supporting domains or other generic domains. Mushin perceives this utilization in the form of a stack of middlewares that resonates well with the Chain-of-responsibility pattern. 


require 'mushin'
Mushin::Stack.new do 
	# core domain 
	# in relevance to the application 
	use  DomainA, {}, {}

	# supporting domains 
	# in relevance to the application 
	use  DomainB, {}, {}
	use  DomainC, {}, {}
	use  DomainD, {}, {}

	# generic domains 
	# in relevance to the application 
	use  DomainE, {}, {}
	use  DomainF, {}, {}
	use  DomainG, {}, {}
end
	

require 'mushin'
class Bot < Mushin::Ext
	def initialize(app=nil, opts={}, params={})
	   @app     = app
	   @opts    = opts 
	   @params  = params 
	end
	def call(env)
           # Inbound maniuplation
	   env[:var] = "new_value_inbound"

	   @app.call(env)

	   #Outbound maniuplation
	   env[:var] = "new_value_outbound"

	end
end

# Testing any Mushin Extenstion goes like this 
state = {:var => "value"}
bot = Bot.new Proc.new {},{},{}
bot.call(state)
